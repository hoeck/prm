\section{Allgemeine Projektbeschreibung}

\subsection{Projektthema}

\paragraph*{}
In dem innerbetrieblichen Projekt "Personalfreier Zugang und Ausleihe in der Bibliothek der FHDW" soll der personalfreie Zugang und die personalfreie Ausleihe von Medien (Bücher/CDs/DVDs/Zeitungen...) in der gerade entstehenden Bibliothek der FHDW Dresden realisiert werden. Vorschläge seitens der Auftraggeber sind die Lösung mittels der RFID-Technologie per Gate oder Terminal. Explizites Beispiel ist die Bibliothek der HTW Dresden.

\subsection{Anwendungsgebiet}

\paragraph*{}
Die Ergebnisse dieses Projekts sind der personalfreie Zugang zu einer kleinen Bibliothek (Annahme: ein Raum mit einer Tür ohne Fenster, Glastür, Brandschutztür oder ähnlichem) und die personalfreie Ausleihe der Medien sowie gleichzeitige Übertragung der Daten an ein schon existierende Lotus Notes System.
Hierbei soll die ID des Nutzers ermittelt werden. Nur für autorisierte Personen soll der Zugang zur Bibliothek möglich sein. Den Medien sollen ebenfalls eine ID zugewiesen werden. Die ID des Nutzers, soll mit der ID des Mediums verknüpft, ans System übertragen werden (Anbindung am Lotus Notes).

\subsection{Projektgröße}

\paragraph*{}
Das Budget ist mit maximal 10000 Euro veranschlagt. Die vorhandenen Ressourcen werden soweit wie möglich genutzt. Personal Computer und Server, sowie ein Internetzugang über Wireless Local Area Network sind als gegeben zu betrachten. Benötigte Mitarbeiter werden für die Zeit des Projektes, soweit möglich, freigestellt.

\subsection{Varianten}

\subsubsection{RFID}

\paragraph*{}
Nachteile:
\begin{itemize}
\item Daten auf den Transpondern können unter Umständen einfach ausgelesen werden.
\item Da die Transponder ständig senden entsteht Elektrosmog.
\item Die Technik ist noch unausgereift und zu störungsanfällig.
\item Es entstehen hohe Kosten für die Technik und für die Transponder.
\item Durch Materialien mit großer Dichte können die Daten oft nicht richtig ausgelesen werden, denn dann werden die elektromagnetischen Felder gestört.
\end{itemize}

\paragraph*{}
Vorteile:
\begin{itemize}
\item Es ist kein direkter Sichtkontakt erforderlich.
\item Die Daten können dezentral auf den Tags erfasst werden.
\item Die Tags können gelesen als auch beschrieben werden.
\item Die gleichzeitige Identifikation mehrerer Gegenstände (anti-collision reading) ist theoretisch möglich.
\end{itemize}

\subsubsection{Laser-Barcodescanner}
\paragraph*{}
Nachteile:
\begin{itemize}
\item Langsame Lesegeschwindigkeit (in Abhängigkeit der Schwingfrequenz des Ablenkspiegels).
\item Erfassen neuer, zweidimensionaler Barcodes nicht möglich.

\paragraph*{}
Vorteile:
\begin{itemize}
\item Hoher Leseabstand möglich
\item Lesen auch unter schwierigen Lichtbedingungen möglich
\item Einfache Anwendung auch für Laien
\end{itemize}

\paragraph*{}
Folgende Gründe sprechen daher für das System der Laser-Barcodescanner. Es ist ausgereift und erprobt, kostengünstig und einfach in Handhabung und Wartung. 

\subsection{Ziele}

\paragraph*{}
Im folgenden sollen die Ziele dieses Projekts stichpunktartig zusammengefasst werden.
\begin{itemize}
\item Ergebnis dieses Projekts ist ein stabiles System. Das System soll sowohl mechanischen als auch informationstechnischen Angriffen standhalten. 
\item Der Zutritt der Bibliothek und die Ausleihe der Medien wird vollständig personalfrei erfolgen.
\item Die Betonung liegt auf der Einfachheit und Kostengünstikeit des Systems. Die Kosten sind mit maximal 10000 Euro  veranschlagt.
\item Der Aufwand für die Beteiligten am laufenden System soll möglichst gering gehalten werden. 
\item Die ID des Nutzers und die ID des ausgeliehenen Mediums wird an Lotus Notes übertragen.
\item Benötigte Hard- und Software werden besorgt und aufgestellt, beziehungsweise installiert.
\end{itemize}

\paragraph*{}
Die folgenden Punkte beinhalten Aufgaben die ausdrücklich \textbf{nicht} zum Projektumfang gehören.
\begin{itemize}
\item Es wird nicht zusätzlich für das Lotus Notes System programmiert. Die Registrierung, Verwaltung und Suche wird von Lotus Notes übernommen. Nur die Anbindung an das schon existierende System wird genutzt oder bei Bedarf erstellt.  Zusätzliche Funktionalitäten der Software sind nicht Bestandteil des Projekts.
\item Die derzeitigen Sicherheitsvorkehrungen bleiben unverändert. Die Räume werden nachts abgeschlossen und die Alarmanlage wird eingeschaltet. Große restriktive Vorkehrungen, wie zum Beispiel eine Sicherheitsschleuse, wird es nicht geben. Es wird somit keine Möglichkeit geben die Bibliothek rund um die Uhr zu besuchen.
